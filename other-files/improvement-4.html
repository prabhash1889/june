June - Improvement Plan #4

One roadmap from two sources: improvement-1 (full code review + engineering foundation) and improvement-2 (BridgeAgent-inspired product direction, source verified). The merge rule: improvement-2 had the right destination but skipped the foundation - its own later phases (unattended runs, watch loops, dictation injection, computer-use) multiply the blast radius of exactly the holes the review found. So: foundation first, decoupling and hands-free next, autonomy last, with the audit log pulled all the way forward because every unattended feature stands on it.

2026-07-19 Phases 10-19 (continues PLAN.md) Now / Next / Later
North star

June is a general local-first voice agent / agentic environment. saple-bridge becomes one optional capability among many, not the core. Every capability is an MCP server, every action routes through the policy gate, every feature respects the privacy modes - and nothing acts unattended without a paused approval and an audit trail.

10

Foundation fixes

~2-3 days

The known holes are closed before any surface grows.

  • 10.1Fail-closed policy: send_to_terminal -> destructive (a newline in data executes commands), assign_task -> expensive; unknown actions default to gated; per-server default-class map.
  • 10.2Transcript off the command line: first JSONL line on the stdin channel the runner already holds; the cmd /C injection path is deleted.
  • 10.3Voice privacy at the boundary: Rust transcribe/synthesize refuse cloud voice under on-device modes.
  • 10.4Claude tool-result correlation: map tool_use.id -> name so batch counts render on the default brain.
  • 10.5Live settings propagation: settings://changed event; widget reloads wake/TTS/privacy without restart.
  • 10.6Key gate only when needed: no OpenAI key demand when the stack doesn't use OpenAI or voice is blocked.
  • 10.7Audit log v1 (pulled forward): structured JSONL per tool call - name, redacted params, class, gate decision, approver, timestamp, turn. Phases 17-19 stand on this.
  • 10.8Small fixes: wake-failure backoff; mp3 mime; dead effort env; duplicate listTools(); UTF-8-safe truncation.
  • 10.9The live mic round-trip check pending since Phase 4: speak, hear, interrupt.
Exit Injection repro cannot execute; a spoken send_to_terminal prompts for approval; settings apply live; batch counts render on Claude; the audit log records a full turn.
11

Persistent agent session + memory

~1 week - the multiplier

June stops being per-utterance amnesiac and per-turn slow.

  • 11.1Resident agent process (agent/serve.ts): JSONL over the existing stdio pair - {run, approve, cancel, reset} in, {text, tool, result, approval, final, error} out, tagged by turn. Respawn with backoff; per-turn watchdog. Deletes the npx/tsx/MCP-connect chain from every turn; MCP connections stay warm.
  • 11.2Conversation memory: one query() held open in streaming-input mode (keeps canUseTool in-loop for Phase 14); fallback stable-cwd session resume (cwd-keyed - pin it). OpenAI-compat brain keeps its messages array. New conversation on ~10 min idle + explicit action. persistSession:false under strict modes. (V2 session API was removed in SDK 0.3.142 - build only on query() + options.)
  • 11.3Real turn cancel: barge-in aborts the in-flight turn. No orphaned spending processes.
  • 11.4Long-term memory: user-editable june-memory.md in the system prompt, written via the memory-tool pattern, path-contained like mcp/files. Settings: "what June remembers" + clear. No vectors.
  • 11.5Latency instrumentation: capture-end -> transcript -> first token -> first audio -> total; P50/P95 in Diagnostics. Target: 800ms median once Phase 12 lands.
Exit "open two claude agents" then "now two codex ones as well" works; second-turn overhead < 300ms before first token; barge-in provably stops token spend; June recalls a preference stated yesterday.
12

Local voice stack

~1-2 weeks

Local-voice and Strict-offline become real configurations instead of "mic off". All inference in-process via ONNX Runtime (the Handy blueprint); models download-on-demand with checksum + progress UI; ~650MB full stack.

  • 12.1Silero VAD v5 (~2MB) replaces the RMS gate for endpointing and barge-in; interruption on N consecutive speech-probability frames.
  • 12.2Local wake word: openWakeWord "Hey June" (~200KB, synthetic-speech training; A/B the LiveKit trainer), gated by Silero. Kills cloud phrase-spotting and its per-burst spend.
  • 12.3Local STT behind the SttProvider seam: Moonshine v2 streaming (245MB, MIT, ~107ms updates) primary; Parakeet-TDT-0.6b-v3 int8 accuracy toggle; whisper-rs acceptable as a third alternate but batch-only - it must not displace the streaming primary. Model-size picker.
  • 12.4Local TTS: Kokoro-82M via Kokoros / kokoro-onnx, reusing the sentence streamer unchanged; verify the phonemizer (espeak-ng GPL) path; Supertonic as alternate.
  • 12.5Semantic endpointing: smart-turn-v3 int8 (~12ms CPU) - finished vs mid-thought.
  • 12.6Streaming transcription UX: live interim transcript; optional brain pre-warm on the interim.
  • 12.7Registry flip: offlineSafe/available true on the new providers - existing privacy enforcement needs zero changes.
Exit Full voice turn (wake -> STT -> Ollama brain -> TTS) with the network disabled; P50 voice-to-voice under ~1s with a fast brain.

14

Hands-free & conversational voice UX

~1 weekneeds 11 + 12

The full loop with zero keyboard or mouse.

  • 14.1Auto-accept with countdown (setting; manual stays default): review card auto-sends after ~3s; any interaction pauses it.
  • 14.2Spoken approvals: repeat-back with exact parameters; strict local phrase matcher (yes/no/confirm/cancel) - never the LLM, never satisfiable by tool output; ~8s timeout = deny, announced. Tiering: expensive = spoken yes OK; destructive/external = click required.
  • 14.3Follow-up mode: mic reopens ~5-8s after each reply, no wake word (opt-in) - with 11.2 this turns June into a conversation.
  • 14.4Backchannel: a brief spoken "on it" when a turn starts tool calls.
  • 14.5Interruption polish: barge-in on Silero speech probability; never interrupt a user mid-pause (smart-turn).
Exit Wake -> command -> spoken approval -> execution -> follow-up, hands never touching the keyboard; a denied spoken approval provably blocks the action.
15

Transcript quality & system-wide dictation

~1 week

June earns daily use even when no agent is needed - improvement-2's best product insight.

  • 15.1Auto-edit pass: cheap LLM pass (selected brain, local-capable) strips filler, fixes punctuation, formats lists - before the review gate; toggle.
  • 15.2Personal dictionary: review-gate corrections auto-add terms; biases the STT prompt and the edit pass; user-editable, persists locally.
  • 15.3Voice snippets: a spoken cue expands to saved text ("insert my intro").
  • 15.4Dictation mode: PTT injects the cleaned transcript into the focused app (enigo/SendInput). Safety rule improvement-2 lacked: injection only as the direct result of a user-held PTT press, never agent-initiated, with a visible indicator while active. The user is the actor; June is the keyboard.
  • 15.5App-aware formatting: focused window title only (no content scraping), opt-in - Slack casual, email formal.
Exit Dictate a messy paragraph into Notepad and a browser field -> clean text; a corrected name sticks next session.

Trust & shipping

~1-2 weeks - absorbs the PLAN Phase 9 tail
  • 16.1Bridge-side one-time approval tokens: mint on June-side approval; bridge verifies action+args+nonce+expiry before executing. Closes the "any local process with the discovery token bypasses the gate" hole.
  • 16.2observe() resume + recovery drills: restart restores the roster from the last acknowledged sequence; full drill matrix (killed June/bridge/agent, duplicate requests, partial batches, expired approvals, provider downtime, upgrade).
  • 16.3Injection hardening codified: tool results are untrusted input (never satisfy approvals); full-parameter display/speech before dangerous acts; pinned MCP versions.
  • 16.4Sidecar packaging: Tauri externalBin - prefer Bun --compile, fall back to node.exe + esbuild bundle as resources; avoid pkg/SEA. Gate: prove the Agent SDK's subprocess spawn (query() e2e) from the compiled binary first. CREATE_NO_WINDOW on Windows.
  • 16.5Installer + auto-update, diagnostics export, accessibility pass, structured redacted logs (extends 10.7).
  • 16.6Latency dashboard: P50/P95 voice-to-voice against the 800ms target, per-stage breakdown from 11.5.
Exit An installable build recovers safely from every drill; a fresh machine goes installer -> spoken command with no dev toolchain.

18

Autonomy: scheduled runs & watch loops

requires 16.4 packaging + 10.7 audit log

June works while you don't - without ever acting beyond its leash.

  • 18.1Scheduler: Rust-side cron launches headless sessions via the bundled sidecar; results in the session log; OS notification on completion.
  • 18.2The unattended rule: approval-required actions pause the run and notify - never auto-approve, no blanket session approvals. The audit log is the reviewable record.
  • 18.3Trigger framework (BridgeAgent's production-watch, generalized): file-watch, localhost webhook, log tail -> investigation session with the trigger payload as context. Payloads are untrusted input - 16.3 rules apply; a webhook body can never approve anything or reach a terminal un-gated.
Exit A daily 9am briefing completes unattended and notifies; a webhook opens a gated investigation whose every action is in the audit log.
19

Missions & platform completion

  • 19.1Missions: a user-stated outcome decomposes into a verifiable task list; sequential sessions per task; progress in a simple task-board UI in both faces.
  • 19.2Composable toolsets: only mission-relevant MCP servers load per session (BridgeAgent's idea - their marketing says 25 integrations, not the cited 57; the concept holds) - keeps the per-turn tool surface lean, which voice latency needs anyway.
  • 19.3Zero saple-* required: saple-bridge is an optional Phase 13 catalog entry; June installs and works on a machine that has never seen saple.
  • 19.4Computer-use MCP: only if a real requirement appears (PLAN §8's discipline), opt-in, destructive-class by default, and only after 16.3 + the audit log are field-proven. It is the top prompt-injection amplifier; Playwright covers the browser subset far more safely.
Exit June runs standalone with no saple installed; a mission completes across sessions with a reviewable board and audit trail.

Top risks (merged)

  1. Agent SDK inside a compiled sidecar (16.4): the SDK spawns its bundled CLI as a subprocess; bundlers can break this silently. Spike the compiled-binary query() e2e early; node.exe-as-resource always works.
  2. Session resume is cwd-keyed (11.2): pin cwd explicitly; test across restarts.
  3. Kokoro phonemizer licensing (12.4): espeak-ng is GPL; use a clean-G2P binding or Supertonic.
  4. Wake-word false triggers (12.2): A/B two trainers against a recorded test set; keep the sensitivity slider; Silero gating.
  5. Unattended runs amplify any gate weakness (18): hence the hard ordering - 10.1's fail-closed default and 10.7's audit log land months before 18 does.
  6. Voice E2E test flakiness: recorded-audio fixtures feeding the Rust pipeline directly + golden transcripts; live-mic checks stay manual but scripted.

Acceptance scenario (the general-purpose upgrade)

  1. Fresh machine, no saple installed: installer -> "Hey June" (fully local) -> "what's on my calendar today" via the Google Workspace entry the user added in settings - answered, hands-free.
  2. "Open five claude agents and four codex agents" (saple-bridge attached as an optional capability): June repeats counts and cost class aloud; the user says "yes"; executed once; counts reported from bridge results; follow-up within 8s, no wake word.
  3. Strict offline (switched live, no restart): full spoken round-trip via local wake/STT/TTS and an Ollama brain, zero network; the files capability still works.
  4. Dictation: PTT into a browser text field -> cleaned, dictionary-corrected text appears; nothing injected that the user didn't speak while holding the key.
  5. A daily 9am briefing completes unattended; a gated action inside it paused and notified instead of running; the audit log shows every action, parameter, and approver.
  6. P50 voice-to-voice < 1s, P95 < 2s; restarting June (or the bridge) restores the same observable state.

June - Improvement Plan #1

A full review of the june voice-agent codebase, two research passes (local voice pipeline tech and agent capability patterns, current as of mid-2026), and a six-phase plan ordered by leverage: close the safety holes, make the agent persistent and conversational, make "local-first" true, make hands-free real, grow capabilities, then ship it with trust.

1Review verdict

The architecture is sound and honestly implemented: the provider seams (Brain / SttProvider / TtsProvider), the frozen bridge contract, and the execution-layer approval gate are all real in code, not just in PLAN.md. But the review found 2 safety holes, 1 security bug, and 4 correctness/UX bugs, plus gaps that define the product today: 100% cloud voice, no conversation memory, multi-second per-turn latency.

What is correctly implemented (verified against source)

Approval gate in the execution layer

Both brains route every tool call through the same ToolGate before dispatch. Fails closed: 120s timeout denies, a closed stdin channel denies, decisions are validated and routed to the owning turn.

agent/claude-brain.ts:45 - agent/openai-brain.ts:140 - agent_runner.rs:388

Path containment in mcp/files

Canonical root via realpath, separator-guarded prefix check, realpath re-check of targets and write parents (symlink defense), read cap. Unit tested; no escape found.

mcp/files/paths.ts - mcp/files/server.ts

Secrets never cross IPC

No get_api_key command exists; keys are read only in Rust at call sites; renderer-facing keychain commands validate the service-name convention at the trust boundary.

src-tauri/src/keychain.rs

Brain privacy enforced at run time

A networked brain is refused under a forbidding privacy mode at the execution boundary, so editing settings.json cannot smuggle a cloud brain past strict-offline.

agent/run-once.ts:130

Cross-window session

Seq-stamped capped event log, replay-then-live with buffering and dedup, StrictMode-safe. A window opened mid-session shows the full conversation.

agent_runner.rs - src/app/AppWindow.tsx

Contract + settings discipline

Frozen error codes/actions with golden JSON and validators; atomic temp-file+rename settings writes; typed coercion; merge-over-raw-bag saves; honest provider registry.

src/contract/ - src-tauri/src/settings.rs - src/lib/providers.ts

What is wrong

S1 - SafetyUngated terminal writes and task assignment; fail-open default

agent/policy.ts:39 classifies any unlisted action as automatic. send_to_terminal writes raw text to a live terminal - a payload containing \n executes whatever precedes it: arbitrary command execution with no human yes. assign_task directs a paid agent to arbitrary work ungated. Every future MCP server's mutating tools run silently until someone remembers to classify them.

S2 - SecurityTranscript passes through cmd /C on Windows (command injection)

agent_runner.rs:214 spawns cmd /C npx tsx run-once.ts "<transcript>". Rust escapes per MSVC rules; cmd.exe does not parse by those rules. A transcript containing quotes/%/& (spoken, mis-transcribed, or typed into the editable review card) can break quoting and execute commands. Explicit cmd /C opts out of Rust's CVE-2024-24576 protection.

S3 - SafetyBridge-side one-time approval tokens not implemented

ApprovalToken is frozen in the contract but never minted or verified. Any local process that reads the discovery record's bearer token can command the bridge with no approval; June's gate only guards June's own path. Known and deferred - but it is the gap between "the brain cannot bypass the gate" and "nothing can".

S4 - SafetyVoice-stage privacy is UI-only

The brain is checked at the execution boundary, but Rust transcribe/synthesize will happily call OpenAI under strict-offline; only the VoicePanel politely disables the mic.

C1 - BugTool results never attach for the default (Claude) brain

claude-brain.ts:95 emits onToolResult(actionOf(""), ...) - always empty, because the SDK's tool_result block carries an id, not a name, and nothing maps it back. The app window correlates by action name, so the "started 5 of 5" batch summaries never render on the default brain.

C2 - BugSettings changes never reach the running widget

VoicePanel loads settings once on mount and the widget window never remounts. Enabling wake word, changing TTS voice, or switching privacy mode requires a full app restart - contradicting Phase 8's live-toggle claim.

C3 - UXHands-free is not hands-free

After "Hey June" + speech, the review card still requires a mouse click on "Send to June". Phase 8's exit ("talk to June without touching the keyboard") is unreachable end to end.

C4 - UXWidget demands an OpenAI key before doing anything

Even under strict-offline where cloud voice is blocked anyway. A local-brain, no-cloud user is locked out of the default face.

MinorSmaller issues

STT provider/model setting is fake (Rust hardcodes whisper-1) - JUNE_BRAIN_EFFORT exported, read by nobody - barged-in turns keep running with no kill switch (unbounded background token spend) - wake listener retries failed transcriptions forever (bad key = silent API spam per speech burst) - audio/mpeg mapped to audio.mp4 - read truncation can split a UTF-8 char - duplicate listTools() per turn.

Defining gaps (deferred by plan, but they are the product today)

G1No conversation memory

Every turn is a fresh process seeing only the current transcript. "Do that again", "and four more", any pronoun across turns - impossible. The single biggest capability gap.

G2No local voice exists

STT, TTS, and wake are all OpenAI cloud; "local-first" is aspirational; local-voice / strict-offline modes just turn voice off.

G3Per-turn spawn latency

cmd → npx → tsx → run-once → npx tsx per MCP server → SDK spawn. Seconds of process spawning before the brain sees a word.

G4 / G5observe() unused - live check pending

The event-resume machinery is bridge-side only; restart recovery is unstarted. And the live spoken round-trip has never been manually verified, so Phases 4-8 exits are formally open.

2Research digest - local voice stack

Key finding: Handy (23k+ stars, MIT) already proves June's exact architecture - Tauri + Rust with Parakeet, Whisper, Moonshine, and Silero VAD running in-process via ONNX Runtime. No Python sidecars, one runtime dependency, ~650MB total model footprint for the full local stack.

3Research digest - agent capabilities & safety

Session memory is nearly free

The Agent SDK persists sessions automatically: continue: true resumes the latest session in the cwd; resume: sessionId targets one. Gotcha: keyed to cwd - pin it. The experimental V2 session API was removed in SDK 0.3.142; build on query() + session options. Streaming-input mode keeps one query() alive across user messages with canUseTool handled in-loop - the enabler for spoken approvals. persistSession:false exists for privacy modes.

Long-term memory: one markdown file

Anthropic's file-backed memory tool (memory_20250818) + a user-editable june-memory.md injected into the system prompt. Skip vector stores at single-user desktop scale.

Spoken approvals (Alexa prior art)

Repeat back all exact parameters, accept a strict yes/no matched by a local phrase matcher - never the LLM, never satisfiable by tool output. Timeout = deny ("silence is not consent" is a flagged attack surface). Highest-destructive tier stays click-only.

Safety consensus (MCP spec 2026, NSA/CISA CSI)

Classify-by-action with risk annotations is where the ecosystem went - June's design is aligned. Tool results are untrusted input (indirect injection is the top attack). Pin MCP server versions (npm typosquatting has shipped rogue MCP servers). Audit-log every tool call, params, and permission decision.

Capability order by maturity

1. SDK built-in WebSearch/WebFetch (config-only for the Claude brain) - 2. Google Workspace MCP (email send = gated external effect) - 3. official Home Assistant MCP (intent-scoped, naturally sandboxed) - 4. lightweight Spotify MCP - 5. Playwright MCP strictly on demand (~100k+ tokens/task). Skip raw Windows input control and computer-use.

Sidecar packaging risk

The Agent SDK spawns its bundled CLI as a subprocess; bundlers can break this silently. Verify query() end-to-end from the compiled binary before committing. Prefer Bun --compile or node.exe + esbuild bundle as resources; avoid pkg / Node SEA (prior art of dev-works-prod-fails).

4Sequencing rationale & risks

  • A before everything: small diffs, closes real command-execution holes, fixes the most visible bugs. Nothing in A blocks on research.
  • B is the multiplier: it fixes latency, enables memory, enables in-loop spoken approvals, and simplifies barge-in - do it before C so local-voice wins aren't masked by spawn overhead.
  • C makes the product match its name; C1/C2 are small - land them first within the phase.
  • D depends on B (in-loop gate) and benefits from C (local wake/VAD); D1 can ship any time after A.
  • E is config-driven and interleaves with C/D; each server is an afternoon plus classification and testing.
  • F last, except the audit log (F3) which is cheap and worth pulling into B.

Top risks

  1. Agent SDK inside a compiled sidecar: the SDK spawns its bundled CLI as a subprocess; bundlers can break this silently. Prove query() from a compiled binary in a spike before Phase F; node.exe-as-resource always works.
  2. Session resume cwd-keying: resume silently starts fresh if cwd differs. Pin cwd explicitly; test across restarts.
  3. Kokoro phonemizer licensing: espeak-ng is GPL. Use a binding with a clean G2P path, or Supertonic.
  4. Wake-word false-trigger quality: synthetic-trained models vary. A/B two trainers against a recorded test set; keep the sensitivity slider; Silero gating.
  5. Voice E2E test flakiness: decide the strategy now - recorded-audio fixtures feeding the Rust pipeline directly plus golden transcripts; live-mic checks stay manual but scripted.

5Acceptance scenario (upgraded)

  1. "Hey June" (fully local) → "open five claude agents and four codex agents" → June repeats counts and cost class aloud → user says "yes" → executed once, counts reported from bridge results.
  2. Follow-up within 8s, no wake word: "assign the third codex agent to fix the auth tests" → resolved by stable id, done.
  3. Switch to strict-offline in settings (no restart) → full spoken round-trip via local STT/TTS and an Ollama brain, zero network.
  4. Pull the network cable mid-turn → June reports the failure honestly; restart June → roster restored via observe() resume; the audit log shows every action and who approved it.
  5. P50 voice-to-voice < 1s, P95 < 2s on the latency dashboard.

June Improvement Plan v2 Phases 10-19

North star: june is a general local-first voice agent. saple-bridge becomes one optional capability among many, not the core. Every new capability is an MCP server, every action routes through the policy gate, every feature respects privacy modes.
Now - build immediately Next - after Now lands Later - loops & autonomy

Prereq: the Phase 9 hardening tail in PLAN.md (sidecar bundling, recovery, logs, installer) runs in parallel; Phases 15/17 need the bundled sidecar.

Now
PHASE 10NOW

Local voice stack

Local-voice and Strict-offline modes become real instead of "mic off".
  • whisper.cpp STT (whisper-rs) behind the SttProvider trait; model-size picker with download-on-demand
  • Silero ONNX VAD replaces energy VAD
  • openWakeWord ONNX "hey june" replaces STT-gated spotting
  • Kokoro local TTS behind existing seam
  • Flip offlineSafe: true on new providers
Exit: full voice turn (wake → STT → Ollama brain → TTS) with network disabled.
PHASE 11NOW

Transcript quality

Dictation output feels professionally edited.
  • Auto-edit LLM pass (filler, punctuation, lists) before the transcript-review gate; settings toggle
  • Personal dictionary: review-gate corrections auto-add terms, bias STT + edit pass, persist
  • Voice snippets: spoken cue expands to saved text
Exit: messy dictation → clean text; a corrected name sticks next session.
PHASE 12NOW

Generic MCP client

Users add capabilities without june shipping them. Key decoupling step.
  • Settings UI: add any MCP server (stdio/URL), per-server enable + offlineSafe flag
  • Unknown tools default to approval-required; user can promote a tool to auto-run
  • saple-bridge-control and mcp/files become just the first two entries
Exit: add a GitHub MCP server from settings, list issues by voice, gate fires on a write.
Next
PHASE 13NEXT

Skills & memory

June gets better at repeated tasks (BridgeAgent's recursive trick).
  • Post-run lesson writer appends a short markdown lesson per run
  • Pre-run recall injects top-k relevant lessons into the system prompt
  • Wire saple-memory MCP as an optional server via Phase 12 (config only)
Exit: the same task done twice uses prior lessons the second time.
PHASE 14NEXT

Screen context grounding

"What does this error on my screen mean" just works.
  • Opt-in observe-class tool: active-window screenshot + local OCR (Windows.Media.Ocr)
  • Local OCR only in Strict offline; screenshots never persisted to logs
Exit: ask about an on-screen error dialog by voice, get a grounded answer.
PHASE 15NEXT

Scheduled runs

June works while you don't.
  • Rust local scheduler (cron) launches headless sessions via the bundled sidecar
  • Results in session log; OS notification on completion
  • Approval-required actions pause the run and notify - never auto-approve
Exit: a daily 9am briefing run completes unattended and notifies.
PHASE 16NEXT

System-wide dictation

June is useful even when the agent isn't needed.
  • PTT injects the cleaned transcript into the focused app (SendInput/enigo), reusing Phase 11
  • App-aware formatting from the focused window title (Slack casual, email formal)
Exit: dictate into Notepad and a browser text field.
Later
PHASE 17LATER

Watch loops

External signals become june investigations (production-watch pattern, generalized).
  • Trigger framework: file-watch, localhost webhook receiver, log tail
  • A trigger opens an investigation task: fresh session with the trigger payload as context (e.g. Sentry webhook → root-cause session)
  • All actions still gated
PHASE 18LATER

Missions

Outcome-level tasking, not prompt-level.
  • Mission = stated outcome; agent decomposes into a verifiable task list, sequential sessions per task
  • Simple task-board UI for progress
  • Composable toolsets: only mission-relevant MCP servers loaded per session
PHASE 19LATER

General agent platform

June stands alone.
  • Opt-in computer-use MCP server: screenshot / click / type, destructive-class by default
  • Zero saple-* dependencies required; saple-bridge is an optional MCP entry
  • Standalone installer + auto-update; plugin surface is MCP only
Deliberately not doing
  • Cloud-side execution ("runs with laptop closed") - against local-first.
  • Stealth/undetectable overlays (Cluely-style) - against honest-by-design.
  • Session-wide blanket approvals - per-action gating stays.
